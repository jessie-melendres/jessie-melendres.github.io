Key Concepts Summary

NumPy:

  • Arrays: Fundamental data structure for numerical computation
  • Vectorization: Operations applied to entire arrays without loops
  • Broadcasting: Operations between arrays of different shapes
  • Matrix Operations: Linear algebra capabilities

Pandas:

  • DataFrame: 2D labeled data structure (like Excel spreadsheet)
  • Series: 1D labeled array
  • Data Selection: Multiple ways to select data (loc, iloc, boolean indexing)
  • Data Cleaning: Filtering, handling missing values, etc.

Important Differences:

  • ravel() vs flatten(): ravel creates views, flatten creates copies
  • iloc vs loc: iloc uses integer positions, loc uses labels
  • Pandas vs NumPy: Pandas for labeled data, NumPy for numerical computation

Outliers and Missing Values
Outliers are observations with 'extreme values', which can affect the robustness of statistical models. Some methods are more robust (e.g. quantile regressions) if outliers are present.


Scatter plots can identify outliers, which are 'far away' from other observations. It is good practice to start your data analysis with descriptive analyses of your variables. The following code can be applied to Pandas Series and the whole DataFrame. Note that we continue our analysis of household data.


Minimum and maximum values provide the range of a variable. Percentiles such as the 25-percentile and 75-percentile determine the interval that captures 50% of all observations.


Outliers are usually defined as values that are +3 standard deviation below or above the average value. What should we do with outliers after they are identified?


Many practitioners drop them from the dataset. However, this can be problematic as we do not know the underlying reason for extreme values.


Reasons include genuine mistakes (e.g. measurement errors) but also events that affect certain variables (e.g. economic shock or policy change). The latter can be modelled using dummy variables as outlined in the chapter on linear regressions.


Before outliers are removed, we suggest to explore the distribution of variables. Variables can be transformed (e.g. log transformation) if negative or positive skewness is present.


For instance, household income tends to exhibit positive skewness, i.e. a few large income values can be observed, with the majority of incomes below the mean. This positive skewness can be addressed by applying a natural logarithm.


NumPy can transform a Pandas Series as well as the Pandas method transform() in combination with Python's lambda functions. Note that lambda functions are unnamed functions in Python. They tend to be simple enough to fit into one line of code.

If transformations or dummies fail to mitigate the impact of outliers, winsorization can be attempted. Winsorization replaces extreme values with the value at the 5-percentile and 95-percentile. Other cut-off points are common. Alternatively, robust methods can reduce or eliminate the effect of outliers on parameter estimates.


Missing values is another common problem in data wrangling. Basically, there are two options: (i) ignore missing values or (ii) try to predict missing values using existing data. In most applications in business analytics, missing data is ignored. To be precise, we adopt list-wise deletion, i.e. the observation (row) is removed from the model if one of the variables has missing values. This approach makes sense if data is easily available (e.g. data downloads).


In some disciplines, data is expensive to collect (e.g. medical trials). Accordingly, data imputation techniques have emerged for various data structures. In this course, we tend to adopt the list-wise deletion approach in line with most of the literature in business analytics.






















Simulations
Apart from analysing actual data, simulated data offers useful insights in many applications. Theoretical models might not predict deterministic outcomes but probabilistic insights can be derived. This is the case in many areas of business analytics. Many asset pricing or risk management models refer to stochastic modelling, where outcomes depend on stochastic processes.


This section illustrates the use of simulations to generate interest rates. The Vasicek (1977) model provides the underlying theory, which describes a continuous-time stochastic process. Time is continuous (in theory); hence, the letter d denotes the change in some variable. For instance, a basic continuous-time stochastic process for describing the dynamics of the short rate ($r$) is given by equation:


$$
  dr = bdt + \sigma dz
$$


Note that $r$ is the short rate, and dr denotes the change in the short rate. The change in time refers to dt or, equivalently, the length of the time interval (for a very small interval of time). Finally, dz denotes a random process, and b is the expected direction of the rate change. The impact of the random process is scaled by $ \sigma $.


The random nature of the change in the short rate stems from the random process $ dz $. The assumptions are that the random process follows a normal distribution with a mean of zero and a standard deviation of one (i.e. is a standardized normal distribution). Furthermore, the change in the short rate for any two different short intervals of time is independent. The expected value of the change in the short rate is equal to $ b $, the drift term.


The Vasicek (1977) model modifies equation  above, assuming that interest rates are mean-reverting. The parameter $ \beta $ may be regarded as the equilibrium level of the short-term interest rate around which it evolves.


When r falls far below (above) its long-term value $ \beta $, the expected instantaneous variation of $r$ is positive (negative). In this case, the short-term rate will tend to move up (down). It will move towards its long-term value quickly when it is far from it and when the parameter a, the speed of return to the long-term mean, is high. Finally, $dW$ refers to a so-called Wiener process (Brownian Motion).
$$ dr = a(y-r)dt + \rho dW $$


How can we simulate interest rates based on equation above? The main trick is to derive a discrete version of equation as time is not continuous in programming. So we note that $ dr \approx \Delta r_{t} = r_{t} - r_{t-1} $. We start by importing Python libraries.

class Rate ():
    def __init__(self, N=1000, dt=0.05, rho=0.001, alpha=10, gamma=0.04 , r0=0.04):
        '''
        Parameters
        -----------------------
        N : Number of steps
        dt : Time steps
        rho : Scale
        alpha : Speed of adjustment
        gamma : Long - term equilibrium
        rO: Current interest rate
        '''
        
        self.N = N
        self.dt = dt
        self.rho = rho
        self.alpha = alpha
        self.gamma = gamma
        self.r0 = r0
    
    def wiener(self):
        '''
        Wiener process
        Returns
        --------------
        Array of realisations of Wiener process
        '''
        
        #Result vector
        out = np.zeros(self.N)
        
        #Initial value
        out[0]= np.sqrt(self.dt) * self.rho * np.random.normal (0, 1)
        
        for j in range(1, self.N):
            out[j] = out[j - 1] + np.sqrt(self.dt) * self. rho * np.random.normal (0,1)
        
        return out

class Rate ():
    def __init__(self, N=1000, dt=0.05, rho=0.001, alpha=10, gamma=0.04 , r0=0.04):
        '''
        Parameters
        -----------------------
        N : Number of steps
        dt : Time steps
        rho : Scale
        alpha : Speed of adjustment
        gamma : Long - term equilibrium
        rO: Current interest rate
        '''
        
        self.N = N
        self.dt = dt
        self.rho = rho
        self.alpha = alpha
        self.gamma = gamma
        self.r0 = r0
    
    def wiener(self):
        '''
        Wiener process
        Returns
        --------------
        Array of realisations of Wiener process
        '''
        
        #Result vector
        out = np.zeros(self.N)
        
        #Initial value
        out[0]= np.sqrt(self.dt) * self.rho * np.random.normal (0, 1)
        
        for j in range(1, self.N):
            out[j] = out[j - 1] + np.sqrt(self.dt) * self. rho * np.random.normal (0,1)
        
        return out
        
    def vasicek(self):
        '''
        Returns
        --------
        Array of simulated interest rates
        '''
        
        #Result vector
        out = np.zeros(self.N)
        
        #Initial value
        out[0] = self.r0
        
        #Wiener process
        w = Rate()
        w = w.wiener()
        
        for j in range(1, self.N):
            out[j] = out[j - 1] + self.alpha*(self.gamma - out[j - 1])*self.dt + w[j]
        return out

Machine Problem
Imagine you are trying to develop a strategy to win at Roulette. The idea is the following. You only bet using 50: 50 bets. Of course, one has to note that they are not quite 50: 50 as the green 0 ensures that the bank has an
advantage. You start with a fixed bet of x, which you can modify at the start of the game.


If you win, you continue with your initial bet. If you lose, you double your bet in the next round. After winning, you adjust your bet back to the initial bet. The idea is that you can recover from any loss using this betting
strategy. Historically, some casinos banned this strategy. As far as we know, this strategy is no longer considered against the rules.


Your task is to build a Python code that can run a simulation of this strategy. Implement the strategy based on betting on even or odd numbers.


Your initial cash position, the number of simulated games, and the initial value of the bet should be model parameters. Your simulation should show the gambler's cash position.
